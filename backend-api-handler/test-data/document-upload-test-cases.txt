Authorization: Bearer <access_token>

================================================================================
HTTP STATUS CODES SUMMARY
================================================================================
201 Created         - Documents uploaded successfully (POST)
200 OK              - Document updated/deleted/retrieved successfully
400 Bad Request     - Validation errors (missing fields, wrong data)
401 Unauthorized    - Invalid/missing token or not booking owner
404 Not Found       - Booking or document doesn't exist
409 Conflict        - Documents already exist (use PATCH to update)

================================================================================

================================================================================
                           USER FLOW SUMMARY
================================================================================

1. INITIAL UPLOAD (POST):
   - User has no documents uploaded
   - UI shows "Upload Documents" button
   - POST /api/v1/bookings/{id}/documents
   - Must include: 1+ payment receipts + passports for ALL travelers
   - Returns 201 Created

2. VIEW DOCUMENTS (GET):
   - User has documents uploaded
   - UI shows "View" button
   - GET /api/v1/bookings/{id}/documents
   - Returns list of all documents

3. EDIT SINGLE DOCUMENT (PATCH):
   - User wants to fix one wrong document
   - UI shows "Edit" button next to each document
   - PATCH /api/v1/bookings/{id}/documents/{documentId}
   - Updates only that specific document
   - Returns 200 OK

4. DELETE SINGLE DOCUMENT (DELETE):
   - User wants to remove a document
   - DELETE /api/v1/bookings/{id}/documents/{documentId}
   - Returns 200 OK

5. RE-UPLOAD ALL (POST after DELETE ALL):
   - If user deletes ALL documents
   - UI shows "Upload Documents" button again
   - POST becomes available again
